nand2tetris pro07 08 HELP

方法

  1. 首先查看是否是只有一个文件，如果是则直接进入步骤3，如果不是进入步骤2
  2. 添加initalSP指令，和call Sys.init 0指令，之后对于每个文件运行下面几步
  3. 将注释全部去除
  4. 重新细分指令(这会使得某些指令的参数量减小)
    • 遇到push segment变为push_segment
    • pop同上（但是同时不改变其他参数的值）
  5. 通过表1(cmd的替换规则)获取cmd要使用的是哪一个汇编模版以及需要替换的元素
  6. 使用模版并替换，同时将模版中其他需要替换的元素也一同替换

function

  1. 定义一个函数 使用function命令 具体干的事情：
  2. 定义代表下面开始运行这个函数，这时要保证栈是空的，但系统栈不可能为空，即此时直接默认栈为空即可
  3. 所做的事情很简单，回忆LCL，它用于储存目前函数的局部变量，由于参数被拉进来，要给局部变量给位置，但此时只是声明，不知道具体的值，所以全部为0
  4. 那么我们具体的做法为，将目前栈顶指针赋给LCL，先给k（局部变量数量）的局部变量位置，那么SP指针增加k个，也即这k个栈中的位置就用于参数的局部变量

call

  1. 在call某个函数时，做的事情即为push一堆东西（目前系统的运行环境），按照书中给出的图示插入即可。
  2. 然后确定几个指针的位置即可，具体的
  3. 在此函数运行之前他的参数已经被push，那么只需要将ARG定位那个位置即可
  4. LCL用于保存局部变量，由于在function处要给他的局部变量空间，目前call的时候并不知道，那么就把LCL置为SP即可。实际实现中这个地方什么都没有做，这是因为要做的事情都在function指令处做完了。
  5. 返回地址，在call汇编的最后放一个标签，将其入栈，相当于保存了call之后的运行位置，return之后能够继续运行。这样整个运行完之后就能回到这个位置
  6. 再调用函数的全部过程中，参数入栈，保留目前环境，重置ARG都不虚过多叙述。对于LCL，call的时候因为不知道局部变量的数量，什么都不做，之后跳转到function开始运行，做的事情为现将SP给LCL，将SP往下移k个位置，k此时已知。这样就成功调用了另一个函数开始运行它。

return

  1. return时 要先保存这么几个东西
  2. 栈返回位置(*ARG)到R13保存，因为此时ARG所指的位置为第一个参数，这个函数运行完之后这个位置实际上是栈指针的位置。
  3. 函数返回值到R14保存，返回值目前位置是栈顶元素。
  4. 返回的地址保存在R15，返回地址之前已经被保存在栈里。
  5. 将其他所有指针从栈中弹出，保存到相应位置就好。
  6. 然后将正确的保存过的值在几个指针中保存即可。
